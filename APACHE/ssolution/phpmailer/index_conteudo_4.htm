  1. Servidor DHCP com autenticação de clientes via hardware  
    Para ingressar na rede interna basta que o microcomputador do usuário possua uma placa de rede padrão ethernet instalada e configurada com protocolo de rede TCP/IP e para obtenção automática do número de IP a partir de um servidor.

A autenticação se dará por meio de verificação do MAC-Address do hardware da placa de rede de cada máquina, sendo este único para cada placa por processo de fabricação o que garante a integridade da informação do microcomputador do usuário.

O serviço de autenticação dos microcomputadores serve basicamente para prover de um modo fácil e seguro a verificação da autenticidade dos equipamentos dos usuários e a distribuição segmentada dos números da rede IP que atenderá aos usuários da rede interna de forma segura e garantindo a privacidade dos dados de cada usuário.

  2. VLAN Manager  
O serviço VLAN é a implementação da segmentação dos ranges de rede de números IP a serem utilizados pelos microcomputadores dos usuários garantindo a individualização das redes com a criação de sub-redes privadas virtuais.

O Servidor DHCP disponibiliza automaticamente os números de IP alocados mediante verificação do MAC-Address da placa de rede para cada microcomputador conectado à rede e para cada placa de rede será fornecido um numero IP de rede interna aonde somente o próprio participará desta sub-rede não permitindo que outros computadores registrem-se no mesmo range de rede criando assim as sub-redes privadas virtuais.

3. FIREWALL
   

Conexões com a Internet são mais perigosas do que possamos imaginar, ao fazer uso desta conexão através de uma rede, esta fica vulnerável às investidas ou meras visitas de pessoas não autorizadas, isto acontece porque em uma conexão permanente a rede torna-se um alvo fácil tendo em vista que o endereço de IP válido na Internet fornecido pelo provedor da conexão fica exposto e serve como porta de entrada para hackers ou quaisquer outros intrusos.

Um Firewall como o próprio nome já diz é uma parede corta-fogo para o seu servidor de conexão, tornando-o mais seguro à possíveis ataques e dificultando que outras pessoas possam entrar na rede interna.

Para implementar o serviço de Firewall adotamos a solução de filtragem de pacotes, controle de conexões às portas de serviço do protocolo de comunicação TCP/IP e monitoramento dos acessos ao servidor, via softwares baseado no sistema operacional Linux.

  4. ROTEADOR  
    A utilização de serviço de Roteador para prover acesso à Internet baseado no serviço NAT portado no sistema operacional Linux* fornece à rede interna uma porta de conexão com o mundo externo. 

O serviço de roteamento não interfere na velocidade da conexão, pois a velocidade interna da rede é definida pelo hub / switch (100 Mbps) que conecta os clientes internos e a velocidade do link de acesso será compartilhada apenas pelo número de usuário que façam uso do serviço ao mesmo instante.

  5.

Servidor PROXY

  
   

O Servidor Proxy serve basicamente para prover um modo fácil e seguro de acesso a Internet para todas as estações de trabalho da rede interna.

Quando um cliente da rede interna, devidamente autenticado, faz uma requisição de acesso à Internet o Servidor Proxy intercepta o pedido e redireciona para a conexão com à Internet como se fosse a própria estação de trabalho do usuário, ao receber a resposta da Internet o Servidor Proxy entrega o conteúdo recebido ao cliente da rede interna que o requisitou.

Além de prover maior segurança no acesso à conexão Internet o Servidor Proxy permite registrar as requisições dos clientes da rede interna e implementar políticas de caches em disco dos sites acessados, tornando os futuros acessos a estes mesmos endereços mais rápidos  pois o Servidor Proxy utilizará o tempo ocioso do Servidor de conexão para manter atualizadas estas cópias em discos mediante quaisquer alterações no próprio site.

  7. Servidor de E-MAIL (SMTP, POP3, IMAP...) com antivírus  
    Atualmente uma das maiores aplicações da Internet é a comunicação via e-mail, pois além de enviar mensagens pode-se anexar arquivos diversos (textos, planilhas, imagens, etc), para usufruir deste recurso faz-se necessário uma conta de usuário de e-mail junto a um servidor de e-mail e / ou de conteúdo de Internet.

Entretanto a segurança para este meio de comunicação ainda é demasiadamente frágil, portanto o envio de e-mails deve-se estabelecer regras de segurança para evitar possíveis vazamentos de informações ou mesmo ser vitimado por vírus ou arquivos nocivos ao sistema operacional do cliente da rede interna.

9. Servidor DNS
O Servidor DNS executa a transformação dos nomes de endereços de sites (URL) em números de IP localizáveis em uma rede de computadores tanto na Internet como em uma Intranet. Este serviço é necessário está habilitado em caso de aplicações de Intranet e / ou Internet hospedado no servidor da rede interna.
